Introduction to Mobile Applications Development

Mobile Operating Systems

  • Android: Open-source by Google, customizable, supports various devices
  • iOS: Apple-exclusive, smooth user interface, multi-touch support
  • Windows Phone: Microsoft integration (discontinued)
  • Others: BlackBerry, Symbian (discontinued)

Benefits of Mobile Apps

  • Convenience, offline functionality
  • Personalization and responsiveness
  • Touch-based interaction
  • Productivity tools and integration

Development Environments (IDEs)

  • Android Studio: Java/Kotlin for Android
  • Xcode: Swift for iOS
  • Visual Studio: Cross-platform (Windows, Android, iOS)

Design Considerations

  • Varying screen sizes and densities
  • Touch-friendly input (minimize typing)
  • Responsive layout
  • Integration with phone features
  • Efficient use of battery and CPU

Android App Development Basics

Application Fundamentals

  • APK: Package containing code, assets, manifest
  • Apps run in sandboxed environment using separate processes

App Components

  • Activities: UI screens
  • Services: Background tasks
  • Content Providers: Manage shared app data
  • Broadcast Receivers: Listen for system-wide events
  • Manifest File: Declares app components and permissions
  • Intents: Messages that activate components

Development Workflow

  • Setup IDE and virtual device
  • Develop code and UI
  • Test using emulator or physical device
  • Build and publish APK

Tools

  • Android SDK: APIs and libraries
  • ADB: Debugging and device communication
  • Gradle: Project build automation

User Interface (UI) Design

UI Components

  • Views: Buttons, text fields, images
  • ViewGroups: Layout containers (LinearLayout, ConstraintLayout)
  • App Bar: Title and navigation
  • Dialogs: Prompts or confirmations
  • Notifications: System-wide alerts
  • Toasts: Short messages shown on screen

Layouts

  • LinearLayout: Aligns elements in a single direction
  • RelativeLayout: Positions elements relative to each other
  • ConstraintLayout: Modern, flexible layout
  • ListView/GridView: Display scrollable lists or grids

Input Controls

  • Buttons: Trigger actions
  • Text Fields: Input from user
  • Checkboxes/Radio Buttons: Multiple/single selections
  • Spinners/Pickers: Choose values or dates from dropdowns

Android Activities & Lifecycle

Activity States

  • Resumed: Foreground and interactive
  • Paused: Partially visible
  • Stopped: Hidden from user

Lifecycle Methods

  • onCreate(): Initialize app components
  • onStart() / onStop(): Activity becomes visible/invisible
  • onResume() / onPause(): Activity gains/loses focus
  • onDestroy(): Cleanup before destruction

Fragments

Modular parts of UI with their own lifecycle.

Common methods: onCreateView(), onPause(), onDestroyView()

Input Events

Event Listeners

Monitor user actions on UI components.

Examples: setOnClickListener, setOnTouchListener

Event Handlers

Define responses to user interactions.

Example: Navigating to another screen after a button click.
