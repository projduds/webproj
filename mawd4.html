Database Systems vs. File Systems

File Systems

Organize data as flat files. However, they are:

  • Time-consuming to manage
  • Inefficient for large data queries
  • Difficult to secure and maintain

Database Systems

Use a Database Management System (DBMS) for structured and secure data storage.

Key Components

  • Hardware: Physical servers and storage
  • Software: DBMS (e.g., MySQL), OS, tools
  • Peopleware: DBAs, designers, users
  • Procedures: Policies for handling data
  • Data: Facts and metadata

Database Models

  • Conceptual Model: What data is represented
  • Implementation Model: How data is structured and stored

Core Building Blocks

  • Entities: Objects (e.g., Student)
  • Attributes: Properties (e.g., StudentID)
  • Relationships: Links (1:1, 1:*, etc.)
  • Constraints: Rules (e.g., Age ≥ 18)

Types of Data Models

  • Relational: Tables (rows and columns)
  • Hierarchical: Tree structure (one parent per child)
  • Network: Graph structure (multiple parents allowed)
  • E-R Model: Visual entity-relationship diagrams
  • Object-Oriented: Combines data and behavior

SQL Basics

SQL (Structured Query Language) is used to manage relational databases.

Main Categories

  • DDL: Data Definition Language – CREATE, ALTER, DROP
  • DML: Data Manipulation Language – SELECT, INSERT, UPDATE, DELETE
  • DCL: Data Control Language – GRANT, REVOKE

Advanced SQL Operations

  • Aggregation Functions: SUM(), AVG(), COUNT()
  • Joins:
    • INNER JOIN: Matches records from both tables
    • LEFT/RIGHT OUTER JOIN: Includes unmatched records
    • FULL OUTER JOIN: All matched and unmatched rows
  • Views: Virtual tables created from queries for simplicity and security

Normalization

Improves data organization by reducing redundancy.

Normal Forms

  • 1NF: Atomic columns; no repeating groups
  • 2NF: Full functional dependency on the primary key
  • 3NF: Removes transitive dependencies
  • Higher Forms: BCNF, 4NF, 5NF – handle complex dependencies

ASP.NET Overview

What is ASP.NET?

ASP.NET is a web development framework for building dynamic web applications on the .NET platform.

Framework Types

  • Web Forms: Visual drag-and-drop UI
  • MVC: Structured, scalable, and testable
  • Web Pages: Simple, script-based web pages

Features

  • Security mechanisms
  • HTML5 & Bootstrap support
  • Built-in ORM via Entity Framework

ASP.NET MVC Structure

  • Model: Data and business logic
  • View: UI representation (HTML, Razor)
  • Controller: Handles user requests and data flow

Benefits

  • Clear separation of concerns
  • Easier to test and maintain
  • URL routing and clean URLs

Data Handling in ASP.NET

Entity Framework (EF)

ORM for interacting with databases using classes (POCOs)

Validation

  • Data Annotations: [Required], [StringLength]
  • ModelState: Validates submitted model data

Forms

Use HTML helpers like @Html.TextBoxFor to bind input fields to model properties

Key Takeaways

  • Databases offer secure, efficient, and organized data handling compared to file systems
  • SQL is foundational for managing relational databases
  • Normalization prevents redundancy and increases consistency
  • ASP.NET MVC supports scalable and maintainable web app development

Introduction to .NET MAUI

Overview

.NET MAUI (Multi-platform App UI) is a cross-platform framework used to build native apps for Android, iOS, macOS, and Windows using C# and XAML. It allows developers to create apps with a single codebase using a “write once, run anywhere” approach.

Key Features

  • Unified single project system for multi-platform development
  • Cross-platform APIs for device features (e.g., GPS, clipboard)
  • Hot Reload for instant UI/code updates
  • Integrated layouts, navigation, and data-binding

Visual Controls and UI Building Blocks

Core Components

  • Application Class: Manages the app's lifecycle (App.xaml, MauiProgram.cs)
  • Pages: Represent screens (ContentPage, TabbedPage, FlyoutPage)
  • Views: UI elements like Label, Button, Image, etc.
  • Layouts: Structure the UI (StackLayout, Grid, FlexLayout)

Navigation

  • Shell: Streamlines navigation with flyouts, tabs, and URI routing
  • FlyoutItem: Side menu item
  • TabBar: Bottom tab navigation component

UI Design with XAML

Basics

  • Uses XML-based syntax to declare UI elements
  • Separates UI from logic
  • Example: <Label Text="Hello" /> maps to new Label() in C#
  • Supports event handling and data-binding

Common Layouts

  • StackLayout: Vertical or horizontal stacking
  • Grid: Structured rows and columns
  • AbsoluteLayout: Uses fixed coordinates
  • FlexLayout: Wraps elements automatically

Data Storage Options

  • Preferences: Store simple key-value pairs (e.g., user settings)
  • File System: Save files within app sandbox (e.g., logs, .txt files)
  • SQLite Database:
    • Use SQLite.NET ORM for object-database mapping
    • Supports async operations via SQLiteAsyncConnection

Web Services and Connectivity

Network Detection

  • Check with Connectivity.NetworkAccess
  • Use Connectivity.ConnectivityChanged for real-time updates

RESTful APIs

  • Use HttpClient for GET, POST, PUT, DELETE
  • Parse JSON using JsonConvert.SerializeObject() / DeserializeObject()

Navigation Patterns

  • Flyout Navigation: Sidebar menu using FlyoutItem and ShellContent
  • Tab Navigation: Bottom/top tabs via TabBar
  • Hierarchical Navigation: Stack-based using NavigationPage

Platform Integration

Device Features

Access native APIs like GPS, accelerometer, clipboard, and file picker via a cross-platform abstraction layer.

App Sandbox

Secure platform-specific file storage using FileSystem.AppDataDirectory

Key Takeaways

  • .NET MAUI enables code reuse across platforms with native performance
  • XAML + C# offer flexible and responsive UIs
  • Supports multiple data storage options for different use cases
  • Integrates REST APIs, Shell navigation, and Hot Reload for rapid development
